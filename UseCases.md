## USE CASES

# USE CASE 1:  Recruiter Wants to Quickly Check CV–JD Match
Primary User: Recruiter
Goal: Quickly understand how well a candidate fits a job.
Flow:
Recruiter uploads a CV and JD.
System extracts skills, experience, and qualifications from both files.
System calculates a match score (e.g., 72% or “Medium Match”).
System highlights matching skills and missing skills.
Recruiter uses the result to decide whether to shortlist or reject.
Outcome: Fast and consistent initial screening.

# USE CASE 2: Candidate Wants to Check Their Fit Before Applying
Primary User: Job Applicant
Goal: Understand how suitable they are for a role.
Flow:
Candidate uploads their CV and JD.
System generates a match score.
System highlights missing skills and areas to improve.
Candidate see result and improves.
Outcome: Candidates apply more confidently and with better-prepared CVs.

# USE CASE 3: Hiring Manager Wants Justification for Shortlisting
Primary User: Hiring Manager
Goal: See reasoning behind why a candidate was shortlisted or rejected.
Flow:
Recruiter uploads CV and JD.
System shows match score and explanation.
Recruiter shows result to hiring manager.
Hiring manager sees:
Key matching skills
Critical missing skills
Experience relevance
Final hiring decisions are supported with clear evidence.
Outcome: Evidence of good decision-making.
